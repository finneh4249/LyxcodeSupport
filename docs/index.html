Common Errors

During the development of a discord.js bot, especially by inexperienced developers, errors often occur. These errors usually appear frequently during the early stages of the development process.

Error
If your console is showing: 

DiscordAPIError: Invalid Form Body
0.permissions: Must be 10 or fewer in length.

This is because in your discord server there are more than 10 roles with the specified permission you used in your command. E.g "ADMINISTRATOR"
The number 0 as showed in this example refers to the first command from top to bottom in your commands folder.
You may also filter out the first 10 roles to avoid the error. This will let 10 roles work but the rest will not have permission to use the command.
Fix
Code by KevinFoged. Replace the line in your command handler where you filter roles to the following: 
 return MainGuild.roles.cache.filter((r) => r.permissions.has(command.permission) && !r.managed).first(10);

Error 

client.commands.set(....)
TypeError: Cannot read property 'set' of undefined
This error usually occurs when the client.commands collection is not defined in your index.js file Fix
Add the following line to your index.js file. 
client.commands = new Collection();


Error 

TypeError: (intermediate value) is not a function
OR 
TypeError: Cannot read properties of undefined (reading 'forEach')
Read which line of code it mentions above the error in the terminal. The error happens because the line most likely starts with a ( or [, and the line of code before it doesn't end with a semicolon. Semicolons are optional, except when they're not. The following video will explain what I stated above with a couple of examples at around 5:17, but I recommend you watch the entire video.
If you are getting this error while following the event handler tutorial, you therefore most likely just forgot a semicolon on the line above it: Fixes
// INCORRECT:
const Table = new Ascii("Events Loaded")
(await PG(`${process.cwd()}/Events/*/*.js`))

 // CORRECT:
const Table = new Ascii("Events Loaded"); // Semicolon to the rescue!
(await PG(`${process.cwd()}/Events/*/*.js`))
OR 
// INCORRECT:
    client.commands = new Collection()
    
    ["Events", "Commands"].forEach(handler => {
        require(`./Handlers/${handler}`)(client, PG, Ascii);
    });
    
    // CORRECT:
    client.commands = new Collection();
    
    ["Events", "Commands"].forEach(handler => {
        require(`./Handlers/${handler}`)(client, PG, Ascii);
    });

Can occur if:
  1. Your variable is not declared/defined.
  2. You are trying to access a variable that's not available in the scope.
For further information:
https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Errors/Not_defined

Error 

SyntaxError: await is only valid in async functions and the top level bodies of modules.
Fix
Make your function an async function by adding async to the beginning of your function declaration (usually your execute function)
